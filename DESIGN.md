# Design System: Compliance Hub

内部合规管理系统 · 设计指南

---

## 设计原则

1. **效率优先** —— 内部工具的核心是让用户快速完成工作
2. **清晰胜于美观** —— 信息层次明确，不追求视觉华丽
3. **温暖的专业感** —— 空状态友好，错误状态有帮助
4. **一致性** —— 相同操作相同反馈，降低学习成本

## 组件规范

### 按钮
```
主按钮：蓝色背景，白色文字，6px 圆角，medium 字重
次按钮：白色背景，蓝色边框，蓝色文字
危险按钮：红色背景，白色文字（仅用于删除、驳回等破坏性操作）
禁用按钮：灰色背景，50% 透明度

高度：36px (标准), 32px (紧凑)
左右内边距：16px
图标 + 文字间距：8px
```

### 表单
```
输入框高度：36px
输入框圆角：4px
输入框边框：1px #D1D5DB
输入框聚焦边框：2px #2563EB

标签：14px, medium, 置于输入框上方
必填标记：红色 * 置于标签后
错误提示：12px, 红色，置于输入框下方

表单项间距：16px
```

### 卡片
```
背景：白色
圆角：8px
内边距：16px
边框：1px #E5E7EB
悬停阴影：0 4px 6px -1px rgba(0,0,0,0.1)
```

### 表格
```
表头背景：#F9FAFB
表头文字：14px, medium, #374151
行高：48px
行边框：1px #E5E7EB (底部)
悬停背景：#F9FAFB
选中背景：#DBEAFE

斑马纹：不使用（现代设计趋势）
```

### 徽章 (Badge)
```
高度：20px
圆角：full
内边距：0 8px
字体：12px, medium

状态色：
- 已通过：绿色背景 #DCFCE7, 绿色文字 #166534
- 待审批：黄色背景 #FEF3C7, 黄色文字 #92400E
- 已驳回：红色背景 #FEE2E2, 红色文字 #991B1B
- 草稿：灰色背景 #F3F4F6, 灰色文字 #4B5563
```

### Toast 通知
```
位置：右上角
宽度：320px
圆角：8px
阴影：lg
自动关闭：3 秒 (错误不关闭)

成功：绿色左边框 4px
错误：红色左边框 4px
警告：黄色左边框 4px
信息：蓝色左边框 4px
```

## 交互状态

### 加载状态
```
- 列表加载：骨架屏 (灰色闪烁动画)
- 按钮加载：spinner + 文字变灰
- 表单提交：按钮禁用 + loading 状态
- 页面切换：顶部进度条 (类似 YouTube)
```

### 空状态
```
图标：大图标 (64px), 灰色
标题：16px, medium, "还没有待办事项"
说明：14px, secondary, "太棒了！所有合规记录都已处理完成。"
操作：主按钮 "[创建新的合规记录]"
```

### 错误状态
```
- 字段错误：输入框红色边框 + 下方错误提示
- 表单错误：顶部错误摘要 + 滚动到第一个错误字段
- 网络错误：全屏错误页 + 重试按钮
- 404：插画 + "页面不存在" + 返回按钮
```

## 响应式断点

```
mobile: < 640px   (单栏布局，汉堡菜单)
tablet: 640-1024px (双栏布局)
desktop: > 1024px  (三栏布局，完整导航)
```

### 移动端适配
```
- 表格：卡片式展示或横向滚动
- 表单：单栏布局，标签置于输入框上方
- 导航：底部标签栏或侧边抽屉
- 按钮：全宽或最小 44px 触摸区域
```

---

## 无障碍要求

```
对比度：正文 AA (4.5:1), 大标题 AA (3:1)
触摸目标：最小 44x44px
键盘导航：Tab 顺序合理，Focus 可见
屏幕阅读器：所有图标有 aria-label
表单错误：aria-describedby 关联错误提示
```

---

## 图标系统

使用 Heroicons 或类似开源图标库

```
尺寸：16px (小), 20px (中), 24px (大)
颜色：继承文字颜色
用途：操作图标 + 状态图标
```

---

## 动画

```
时长：150ms (快), 300ms (标准), 500ms (慢)
缓动：ease-out (进入), ease-in (离开)
用途：状态切换、展开/收起、页面过渡
```
